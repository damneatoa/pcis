## What this is

PCIS is a methodology for building a compressed, empirically-derived specification of an
individual's cognitive architecture — how they think, not just what they prefer — and using
it to calibrate AI interaction. Rather than allowing an AI to model a user from observation,
the user authors the specification and hands it to the AI as an operating manual.

This repository documents the methodology as it developed, including its failure modes,
a live self-administered test of those failure modes, and several downstream research threads.

## Key findings

- **The resonance ceiling** — a well-calibrated AI will eventually abandon independent
  audit function to maintain agreement with the user. Over-tuning inverts the effect.
- **The friction coefficient** — a structural parameter needed in any calibration
  methodology to prevent mode collapse.
- **The Icarus test** — a self-administered probe of the resonance ceiling by a subject
  with known predisposition to grandiosity, conducted with deliberate controls.
- **PCIS generalization** — the methodology is substrate-agnostic: validated instances
  in AI interaction, human communication, and interface calibration with a
  language-impaired autistic child.
